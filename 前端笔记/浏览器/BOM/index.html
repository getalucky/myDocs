    BOM

    概念

    浏览器对象模型 (Browser Object Model ) 首字母缩写。

    BOM 对象是浏览器的顶级对象，对应对象名是 window 。

    像我们之前学习的 document 文档对象其实都只是 window 顶级对象下的其中一个对象而已。

    也就是说：DOM 其实也是属于 BOM 的一部分而已。

    localStorage 对象(本地存储)

    • localStorage 用来管理浏览器本地存储数据的，本地存储数据是直接保存到 浏览器 中的。
    • localStorage 属于永久存储，浏览器关掉后，数据其实还在本地。
    • localStorage 数据可以在下一次打开页面的时候重新获取到。

    方法

    方法名 参数 描述
    getItem() 一个字符串的参数(键名称) 获取一条数据，返回字符串数据
    setItem() 传两个参数：(键名称 , 数据) 直接保存到浏览器，参数二自动隐式转换成字符串
    removeItem() 一个字符串的参数(键名称) 移除一条数据
    clear() 无需传参 清空所有数据
    • 键名称和数据都是字符串
    • 在本地存储中，数据都是以键值对的方式存储的，而且存储的都是字符串。
    • 本地存储的数据只存储在相关的页面，其他页面获取不到。
    • getItem()通过键名称获取相对应的数值
    • getItem有返回值，返回字符串格式数据，一般还要配合 JSON.parse() 转换成对象。
    • setItem 一般使用的时候需要配合 JSON.stringify() 先把对象转换成字符串后存储。

    BOM 其他对象

    location URL 对象

    location
这个对象中包含了url地址的相关信息

location.search 通过search这个属性可以获取url地址栏后面的参数
    属性和方法 描述 备注
    href 设置或返回完整的 URL 常用
    search 设置或返回从问号 (?) 开始的 URL（查询部分）。 要获取页面参数时使用，便捷
    reload() 刷新页面，重新加载当前文档。 true 参数传入，绕过缓存，强制刷新

    navigator

    navigator
这个对象中包含了浏览器相关信息，版本号、浏览器类型、移动端/PC端

    Screen

    Screen:  屏幕
这个对象中包含的屏幕显示的相关的信息，屏幕的宽度、高度………………

    history

    History
这个对象中包含了浏览器的浏览器的历史记录

    Window对象补充

    window.onload

    这个事件会等到页面资源加载完之后才去触发

    window.onload = function(){}

window.addEventListener("load",function(){})

    window.getComputedStyle(元素)

    在window对象中提供一个方法来获取元素的计算样式

    var res = window.getComputedStyle(元素)

console.log(res.width);     //res.width的值是只读的

    元素.style.属性 只能获取行内样式，所以要用getComputedStyle获取外联样式。

    定时器

    多次定时器

    设置多次定时器

    const timerId = setInterval(定时执行的函数地址, 毫秒时间);

    清除多次定时器

    clearInterval(定时器启动时候的编号`timerId`);

    注意事项：

    • 调用 setInterval() 方法的时候，定时器启动后有返回值，返回定时器的编号，一般用变量接收用于以后清除使用。
    • 定时器返回的值称为定时器id，每个定时器的id都是不一样的
    • 多次定时器启动后是一直运行，如果不清除或者关掉页面，定时器会一直保持运动。

    单次定时器

    设置单次定时器

    const timerId = setTimeout(定时执行的函数地址, 毫秒时间);

    清除单次定时器

    clearTimeout(定时器启动时候的编号`timerId`);

    注意事项：

    • 单次定时器 Timeout 翻译超时的意思，时间一到执行函数内的代码，并且清除掉自己，后续不再执行。
    • 可通过 clearTimeout 在时间还没到的之前提前主动清除。
