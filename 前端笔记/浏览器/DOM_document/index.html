    DOM_document

    DOM（Document Object Model）：文档对象模型

    获取页面元素

    获取 body 元素(不推荐)

    var body = document.body;  
console.log(body);
console.log(head); //head也可以

    body 是页面内主体标签，有特殊性，可快速获取，但是 div 这类标签就不行。

    根据 id 名称获取元素

    var box = document.getElementById('box');
console.log(box);

    注意：由于id名具有唯一性，部分浏览器支持直接使用id名访问元素，

    <div id = "box"></div>

    console.log(box);  // 输出 ： <div id = "box"></div>

    但不是标准方式，不推荐使用。

    根据标签名获取元素 (不推荐)

    var inputs = document.getElementsByTagName('input');
console.log(inputs);

    注意：通过标签名获取到的数据是伪数组，不管标签是一个还是多个，返回的都是伪数组。

    根据标签选择器获取元素(常用)

    1. querySelector

    2. 只拿第一个满足条件的

    3. 如果没有满足的条件，返回一个空

    4. querySelectorAll

    5. 选择所有满足的元素，返回一个伪数组

    6. 如果没有满足的条件，返回一个空的数组
    <div id="box"></div>

    let box = document.querySelector('#box');  //<div id="box"></div>
    • css里选择器里怎么写，querySelector('选择器')的括号里就可以怎么写

    标签选择器在选择类和Id时要加上.和#

    补充的查找方式

    1. getElementsByClassName (不推荐)
    2. getElementsByName (不推荐)

    动态创建元素createElement

    createElement() 创建元素

    document.createElement('标签名')

    • 在内存创建新的标签，但是不会显示在页面中

    • 如果要在页面中显示，还需配合 JS 负责添加到页面的方法才可以

    事件

    事件三要素

    • 事件源: 被触发事件的元素，如 当按钮被操作
    • 事件类型: 事件的触发方式，如 鼠标点击 或 键盘输入
    • 事件处理函数: 事件触发后要执行的代码，如 函数内让 body 变背景色

    事件的基本使用

    // 1. 查找事件源
var box = document.getElementById('box');
// 2. 给事件源添加点击事件
box.onclick = function() {
    // 3. 事件处理函数内部执行的代码
    console.log('代码会在box被点击后执行');  
};

    事件绑定两种写法

    绑定写法1

    通过on系列属性添加事件

    事件源.on事件类型 = 事件处理函数;

    使用on的注册方式，后面同类型的事件会覆盖前面的事件

    on的方式注册的事件：语法简介，可读性好

    绑定写法2

    事件源.addEventListener('事件类型', 事件处理函数);

    使用addEventListener的方式注册，相同类型的事件不会被覆盖，可以对同一元素注册多个相同的事件。

    事件解绑

    解绑写法1

    事件源.on事件类型 = null;

    解绑写法2

    事件源.removeEventListener('事件类型', 事件处理程序`fn`)

    使用removeEventListener方式解绑事件，事件处理程序不能使用匿名函数(可以将函数抽离出来)。
