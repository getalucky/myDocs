    移动端

    viewport视口布局

    在移动端里面提出了一个视口的布局

    只需要在页面的头部，加上这样的一段代码 —— 标准写法 —— 标准视口

    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximun-scale=1, minimun-scale=1,user-scalable=no" >

    name="viewport" - 代表着这段代码是设置视口的

    width=device-width - 设置视口的宽度 = 设备宽度

    initial-scale=1.0 - 初始的缩放比例是1.0

    maximun-scale=1 - 设置最大的放大倍数

    minimun-scale=1 -- 设置最小的缩小倍数

    user-scalable=no -- 是否允许用户进行缩放

    媒体查询

    1. 在css中写入媒体查询代码 @media screen and (width:xxx) 除了xxx之外，其他都是关键字
    2. 只有当屏幕的宽度满足 width:值 条件时（主流使用宽度），才会去加载里面的css代码

    特性

    可以根据屏幕的宽度，高度等参数进行查询。这些参数就叫做媒体特性。（主流使用宽度）

    • width-宽度
    • min-width 最少宽度
    • max-width 最大宽度
    • height 高度
    • min-height 最少高度
    • max-height 最大高度
    • aspect-ratio 视口的宽高比
    • min-aspect-ratio 最少宽高比
    • max-aspect-ratio 最大宽高比
    @media screen and (width:值) {
  你希望在某个宽度下实现的效果的css代码
}

@media screen and (max-width:值){
  你希望屏幕的宽度小于等于指定的值的情况下的css代码
}

@media screen and (min-width:值){
  你希望屏幕的宽度大于等于指定的值的情况下的css代码
}

@media screen and (min-width:值) and (max-width: 值){
  你希望屏幕的宽度介于一个区间的情况下的css代码
}

    关键字

    • not 取反

    • only 实现更好的兼容

    • and 连接（最常用）

    • or 或者

    ```css / 取反 屏幕宽度不为800px的时候 被选中 / @media not screen and (width:800px){ body{ background-color: red; } } / only 更好的兼容 屏幕为800px的时候被选中 / @media only screen and (width:800px){ body{ background-color: red; } }

    /* or 代码中体现为 逗号(,)  意思屏幕为800px或者600px的时候被选中 */
  @media screen and (width:800px),screen and (width:600px) {
    body{
      background-color: green;
    }
  }

    ```

    媒体类型

    媒体类型可以理解为不同的设备，如 打印机，计算机的屏幕等。

    可以加载不同的媒体类型，一般是 screen 使用得最多

    • all 匹配所有的媒体类型 默认值
    • screen 匹配 计算机显示器(浏览器宽度)
    • print 匹配打印机设备

    rem

    1rem = 根元素的font-size大小

    html {
    font-size: 20px;
}

.box {
    width: 1rem;        /*宽度为20px*/
}
