    条件渲染 分支

    v-if

    <h1 v-if="判断">Vue is awesome!</h1>

    v-if 是一个指令，所以必须将它添加到一个元素上。

    • 但是如果想切换多个元素呢？此时可以把一个 <template> 元素当做不可见的包裹元素，并在上面使用 v-if。最终的渲染结果将不包含<template> 元素。

    html <template v-if="判断"> <h1>Title</h1> <p>Paragraph 1</p> <p>Paragraph 2</p> </template>

    v-else

    <div v-if="Math.random() > 0.5">
  Now you see me
</div>
<div v-else>
  Now you don't
</div>

    v-else 元素必须紧跟在带 v-if 或者 v-else-if 的元素的后面，否则它将不会被识别。

    v-else-if

    同 v-else条件差不多

    v-show

    <h1 v-show="ok">Hello!</h1>

    不同的是带有 v-show 的元素始终会被渲染并保留在 DOM 中。v-show 只是简单地切换元素的 CSS 属性 display。

    注意：

    • v-show 不支持 元素，也不支持v-else`。
    • v-if 也是惰性的：如果在初始渲染时条件为假，则什么也不做——直到条件第一次变为真时，才会开始渲染条件块。
    • v-show 就简单得多——不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。

    列表渲染 循环

    v-for

    对于数组

    我们可以用 v-for 指令基于一个数组来渲染一个列表。

    <ul id="example-1">
  <li v-for="item(循环的每一项) in data里的键名">
    {{ item.message }}
  </li>
</ul>

    var example1 = new Vue({
  el: '#example-1',
  data: {
    items: [
      { message: 'Foo' },
      { message: 'Bar' }
    ]
  }
})
    • v-for 还支持一个可选的第二个参数，即当前项的索引。

    ```html

    • {{ parentMessage }} - {{ index }} - {{ item.message }}

    ```

    对于对象

    v-for 来遍历一个对象的属性。

    <ul id="v-for-object" class="demo">
  <li v-for="value in object">
    {{ value }}
  </li>
</ul>

    new Vue({
  el: '#v-for-object',
  data: {
    object: {
      title: 'How to do lists in Vue',
      author: 'Jane Doe',
      publishedAt: '2016-04-10'
    }
  }
})

    也可以提供第二个的参数为 property 名称 (也就是键名)：

    <div v-for="(value, name) in object">
  {{ name }}: {{ value }}
</div>

    还可以用第三个参数作为索引：

    <div v-for="(value, name, index) in object">
  {{ index }}. {{ name }}: {{ value }}
</div>

    事件处理

    行内事件

    • 可以用 v-on 指令监听 DOM 事件，并在触发时运行一些 JavaScript 代码。
    <button v-on:click="方法名">Add 1</button>

    var example2 = new Vue({
  el: '#example-2',
  data: {
    name: 'Vue.js'
  },
  // 在 `methods` 对象中定义方法
  methods: {
    greet: function (event) {
      // `this` 在方法里指向当前 Vue 实例
      alert('Hello ' + this.name + '!')
      // `event` 是原生 DOM 事件
      if (event) {
        alert(event.target.tagName)
      }
    }
  }
})

// 也可以用 JavaScript 直接调用方法
example2.greet() // => 'Hello Vue.js!'

    方法函数放在methods对象里

    • 除了直接绑定到一个方法，也可以在内联 JavaScript 语句中调用方法：
    <div id="example-3">
  <button v-on:click="say('hi')">Say hi</button>
  <button v-on:click="say('what')">Say what</button>
</div>

    new Vue({
  el: '#example-3',
  methods: {
    say: function (message) {
      alert(message)
    }
  }
})

    点击事件的方法传了参数，原来默认有的事件对象就消失了。

    • 在内联语句处理器中访问原始的 DOM 事件。可以用特殊变量 $event 把它传入方法
    <button v-on:click="warn('Form cannot be submitted yet.', $event)">
  Submit
</button>

    // ...
methods: {
  warn: function (message, event) {
    // 现在我们可以访问原生事件对象
    if (event) {
      event.preventDefault()
    }
    alert(message)
  }
}

    • result属于计算属性，computed里的计算属性可以访问this

    • computed里的计算属性还可以观察和响应 Vue 实例上的数据变动

    js var vm = new Vue({ el: '#demo', data: { firstName: 'Foo', lastName: 'Bar' }, computed: { fullName: function () { return this.firstName + ' ' + this.lastName } } })

    fullName计算属性可以根据 firstName 和 lastName 的变化做出实时响应

    注意点：

    • computed的属性去进行v-model绑定，没法实现双向绑定

    watch

    监听属性 watch也能实现根据 firstName 和 lastName 的变化做出实时响应

    js var vm = new Vue({ el: '#demo', data: { firstName: 'Foo', lastName: 'Bar', fullName: 'Foo Bar' }, watch: { firstName: function (newValue,oldValue) { this.fullName = val + ' ' + this.lastName }, lastName: function (val) { this.fullName = this.firstName + ' ' + val } } })

    watch 里的方法有两个参数，第一个是属性的新值，第二个是旧值。

    组件

    通过Vue.component()方法创建一个组件(全局组件)

    • Vue.component()中第一个参数是组件名，必须的，不能使用大写字母。

    • template属性是第二个参数对象中必须的。

    组件里面的属性

    data

    一个组件的 data 选项必须是一个函数

    data: function () {
  return {
    // 定义的data数据
    count: 0
  }
}

    如果 Vue 没有这条规则,可能会影响到其它所有实例。因为组件具有复用性，可以调用多次,作者在设计组件时候为了组件内部的数据都是独立。

    除了data的定义不一样其他都跟原来的一样

    Prop 传递数据

    Prop 是你可以在组件上注册的一些自定义 attribute。当一个值传递给一个 prop attribute 的时候，它就变成了那个组件实例的一个属性

    Vue.component('blog-post', {
  props: ['title'],
  template: '<h3>{{ title }}</h3>'
})

    一个组件默认可以拥有任意数量的 prop，任何值都可以传递给任何 prop。在上述模板中，你会发现我们能够在组件实例中访问这个值

    一个 prop 被注册之后，你就可以像这样把数据作为一个自定义 attribute 传递进来：

    <blog-post title="My journey with Vue"></blog-post>
<blog-post title="Blogging with Vue"></blog-post>
<blog-post title="Why Vue is so fun"></blog-post>

<blog-post title="My journey with Vue"></blog-post>

<!-- 传入一个数字 -->

    2. 通过name传递参数，用params拿数据

      ```js //A页面: this.$router.push({ name: '页面B', params: { data: '我是要传递的参数' } })

      //B页面拿到传来的值: this.data = this.$route.params.data // 拿到上个页面传来的数据 2. 通过`path`传递参数，**用query拿数据**js //A页面: this.$router.push({ path: '/页面B', query: { data: '我是要传递的参数' } })

      //B页面拿到传来的值: this.data = this.$route.query.data // 拿到上个页面传来的数据

    get请求

    const axios = require('axios');

axios.get('/user?ID=12345')
  // 请求成功
  .then(function (response) {
    // handle success
    console.log(response);
  })
  // 请求失败
  .catch(function (error) {
    // handle error
    console.log(error);
  })
  // 总会运行这个方法
  .finally(function () {
    // always executed
  });


// 使用params传参
axios.get('/user', {
    params: {
      ID: 12345
    }
  })
  .then(function (response) {
    console.log(response);
  })
  .catch(function (error) {
    console.log(error);
  })
  .finally(function () {
    // always executed
  });

async function getUser() {

    console.log(response);
  } catch (error) {
    console.error(error);
  }
}

    入口文件main.js配置globalProperties挂载全局方法对象

    main.js

    import { createApp } from 'vue'
import App from './app.vue'
import axios from 'axios'

const instance = axios.create({
    baseURL: 'http://localhost:3000/posts'
});

const app = createApp(App);
app.mount('#app');
app.config.globalProperties.$http = instance;

    使用getCurrentInstanceAPI获取全局对象

    // 组件文件中

import { getCurrentInstance } from 'vue'

// 方法1
const currentInstance = getCurrentInstance()
const { $http } = currentInstance.appContext.config.globalProperties;

console.log($http().then((res)=>console.log(res.data)))

// 方法2
const { proxy } = getCurrentInstance()
console.log(proxy.$http().then((res)=>console.log(res.data)))
