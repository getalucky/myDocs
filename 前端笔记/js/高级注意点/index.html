    js进阶注意点

    原型

    原型基本概念

    Javascript 规定，每一个函数都有一个 prototype 属性，指向另一个对象。 这个对象的所有属性和方法，都会被构造函数的实例继承。

    每个实例对象都有它相对的原型

    这也就意味着，我们可以把所有对象实例需要共享的属性和方法直接定义在 prototype 对象上。

    Function.prototype 不是指向一个原型对象而是指向一个函数

    构造函数、实例、原型三者之间的关系

    构造函数：构造函数就是一个函数，配合new可以新建对象。

    实例：通过构造函数实例化出来的对象我们把它叫做构造函数的实例。一个构造函数可以有很多实例。

    原型：每一个构造函数都有一个属性prototype，函数的prototype属性值就是原型。通过构造函数创建出来的实例能够直接使用原型上的属性和方法。

    __proto__

    任意一个对象(包括原型对象)，都会有__proto__属性，这个属性指向了构造函数的prototype属性，也就是原型对象。

    获取原型对象：

    • 通过构造函数.prototype可以获取
    • 通过实例.__proto__可以获取（隐式原型）
    • 它们指向了同一个对象构造函数.prototype === 实例.__proto__

    注意：__proto__是浏览器的一个隐藏（私有）属性，IE浏览器不支持，不要通过它来修改原型里的内容，如果要修改原型中的内容，使用构造函数.prototype去修改

    原型对象添加方法

    一般在原型对象添加方法不添加属性

    Array.protopyte.getMax = function(){ 
    //获取数组最大值的功能代码
}

    原型对象的constructor属性

    默认情况下，原型对象中只包含了一个属性：constructor，constructor属性指向了当前的构造函数。

    检测对象分类 instanceof

    原因：因为不同分类的对象，能调用的方法时不一样的

    __proto__有兼容性的问题

    价值：用于检测实例对象的分类

    原理：判断该构造函数的prototype属性是否在实例对象的原型链上，如果在返回true，否则返回false

    console.log(arr instanceof Array)   //判断arr是否是Array这类的对象

    typeof 检查基本类型数据(Boolean,字符串，数值等)和函数类型的类型。 instanceof 检验复杂类型的对象分类(对象，数组，函数)

    原型链概念

    任何一个对象，都有原型对象，原型对象本身又是一个对象，所以原型对象也有自己的原型对象，这样一环扣一环就形成了一个链式结构，我们把这个链式结构称为：原型链。原型对象的终点是null（Object原型对象的原型对象就到了终点）

    属性查找原则

    如果是获取操作

    1. 会先在自身上查找，如果没有
    2. 则根据__proto__对应的原型去找，如果没有
    3. 一直找到Object.prototyp，如果没有，那就找不到了。

    属性设置原则

    只会修改对象自身的属性，如果自身没有这个属性，那么就会添加这个属性，并不会修改原型中的属性。

    Function

    1. 函数表达式 let fn1 = function (){ }
    2. 函数声明function fn2 () { }
    3. Function构造函数 let fn3 = new Function() （少用）

    Function.prototype常用成员

    • call：调用函数，改变this指向，设置函数内部this对象的值

    js Person.call(obj(对象),name(值),age(值),...)

      1. 把构造函数当普通函数执行
      2. 把构造函数内部的this设置成了传入的对象
    • obj 传给this，其他的实参传给构造函数的形参。

    • 参数较少时，推荐使用

    • apply：调用函数，改变this指向，设置函数内部this对象的值

    • 跟call不同，apply 最多只能传递两个参数

      1. 第一个为函数调用时的this对象
      2. 第二个是多个值组合的数组

    • 如果参数存放在数组中，推荐使用可以和 arguments 配合使用

    • bind：重新指向this，返回一个新的函数，不调用函数。

    • 使用场景: bind可以用于 定时器中，定时器函数中的this指向了window

    • bind不调用函数，返回新的函数，但是绑定函数内部的this和实参

    • 以上方法在Function原型对象上

    • call、apply调用的函数把第一个传入的参数作为this

    所有的函数底层都是通过new Function() 创建实例的

    继承

    根据原型链达到继承

    要注意的两行代码

    Student.prototype = new Person();   //为了让Student.prototype继承Person的方法，使Student的原型对象等于Person的实例对象

Student.prototype.constructor = Student;    //覆盖Student.prototype后失去constructor属性，需要重新设置将Student原型对象的constructor属性重新指回Student构造函数
    • 当方法写在Person.prototype的时候，Student通过继承可以使用该方法。

    组合继承

    借用构造函数继承 + 原型链继承

    • 属性用构造函数继承
    • 使用call() apply() bind() 方法 （具体使用参考Function块）
    • 方法用原型链继承
    //创建Person构造函数
function Person (name,age) {
    this.name = name;
    this.age = age;
}

//在Person的原型对象上添加方法
Person.prototype.sayHi = function(){console.log(`${this.name}能sayHi`)}

//创建Student构造函数
function Student (name,age,sex) {
    //属性继承
    Person.call(this,name,age);
    //在继承属性后在添加其他属性
    this.sex = sex;
}

//通过原型链将Student.prototype与Person的实例对象连接到到一起，可以使用Person原型对象上的方法
Student.prototype = new Person();
Student.prototype.sonstructor = Student;

    类(ES6新增)

    作用：类语法用于创建对象（低版本不支持），在(ES6)之前创建对象是通过构造函数实现

    class Person {
  //constructor是构造函数的意思
  //Person的构造函数
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  //以下书写的方法，都会自动添加到原型对象上
  //person原型对象上的方法
  sayHi() {
    console.log("hello, 我是" + this.name);
  };
}
    • 创建对象语法： class 类 {} 。调用：new 类 { }
    • 添加属性，在constructor(){ }里面添加
    • 添加方法，直接在类里面添加

    类的本质就是构造函数，是ES6新增的

    类的继承

    语法： class 子类 extends 父类{}

    • 子类自动继承父类的属性和方法

    注意点：

    • 子类如果要使用constructor构造器，优先调用super()，表示父类的构造器
    class Student extends Person {
    constructor(参数){
        //继承后的构造函数内部必须先调用super()
        super(参数);
        //后面添加自己的私有属性
        this.属性 = 值;    //查找属性，子类有用子类的，没有往父类查找，可理解子类覆盖父类
    }
}

    创建变量和常量

    • var
    • 会进行预解析，正因为如此，会产生一些问题
    • var定义的变量都是全局变量
    • let
      1. 不会有预解析
      2. 在同一作用域下面不允许有重复声明
      3. 拥有块级作用域
    • const
      1. 用于声明常量
      2. 初始化的时候就要求赋值
      3. 剩余的特点和let一样
    • 本质是将内存地址赋值给定义的变量，并且不允许修改内存地址。但是可以通过一些不改变内存地址的方法改变里面的值。

    作用域

    全局作用域

    内联、外链都能访问，需要注意引入位置

    包含的内容：

    • window对象上的成员：document对象 、localStorage对象、 setInterval、 setTimeout
    • script级声明的量 、 let const声明的量

    变量声明和function函数声明会添加到window对象上

    局部作用域(函数作用域)

    作用域只在函数内部，在调用函数的时候才生成

    arguments、this是 自带的

    块级作用域(ES6新增，利用let、const实现)

    利用let、const形成块级作用域

    for(leti=0;i<5;i++ ){ }

{letaa = 10; }
    1. 块级作用域内声明的变量不会污染到外部
    2. 变量/常量的内存回收机制( 垃圾回收机制)
    3. var 定义的变量没有块级作用域的说法

    JS内存管理机制

    1. 变量/常量使用完毕后会进行内存回收机制

    2. 函数作用域的活动对象在函数运行完毕后回收内存空间

    3. 块级作用域在运行完毕后，也会回收其内存空间

    4. 全局变量/常量并不会回收其内存空间中所以工作用尽量不要使用全局变量

    闭包

    闭包指的是：能够访问另一个函数作用域的变量的函数

    形成条件：至少两个函数，嵌套关系，内部函数需要访问外部函数的变量。闭包只针对函数而言

    作用：

    1. 闭包可以让你从内部函数访问外部函数作用域。
    2. 让这些变量始终保存在内存中，不会随着函数的结束而自动销毁。（会造成内存泄漏）

    注意点：

    1. 闭包只能取得包含函数中任何变量的最后一个值。闭包所保存的是整个变量对象。

      闭包理解所需知识点

    2. 作用域链 函数作用域，在函数写好了，这个函数的作用域就定下来，作用域链也跟着定下来

    3. 活动对象（局部变量）

    执行函数时，由活动对象管理函数内部的局部变量/常量

    • 垃圾回收

    变量/常量使用完毕后会进行内存回收(垃圾回收)，没使用完就会被保留下来

    浅克隆

    // 浅克隆
        const obj1 = {
            a: "aa",
            b: "bb"
        };
        const obj2 = {};
        for (let key in obj1) {
            obj2[key] = obj1[key];
        }
    • 浅克隆: 克隆出来的对象和源对象不同内存地址
    • 浅克隆缺陷:两个对象之间并没有完全独立。
    • 对象的成员是同一个存储空间

    深克隆

    • 深克隆:克隆出来的对象和源对象其包括子对象的内存地址都不同

    很多JS库的对象深克隆其实底层是通过递归实现的深克隆的原理:

    1. 封装一个深克隆函数

    2. 函数内部克隆对象的属性

    3. 判断其子属性是否也是对象，如果是对象，继续调用深克隆克隆子对象
