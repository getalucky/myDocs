    NodeJs

    注意事项:

    • NodeJS环境中没有window对象，没有document对象(因为这些是浏览器的功能)
    • NodeJS只允许运行JS代码，所以这里也不需要关系HTML 结构和CSS样式
    • 在node中，不能写任何的dom和bom相关操作的代码，否则报错。

    nodeJs支持的对象

    1. Array
    2. StringMath
    3. Function
    4. Date

    5. Math

    6. NodeJS还支持定时器的语法

    7. 在浏览器环境下定时器是属于window 对象的
    8. 在NodeJS环境下定时器是属于global对象的
      • NodeJIS全局对象是global

    内置模块

    NodeJS把功能都以模块的方式拆解，按需导入。

    导入模块

    let fs = require('模块');

    fs模块（读写文件）

    writeFile

    语法: fs.writeFile('文件地址', 数据, function(err){ }); //异步
    • 如果写入的路径不存在，回调函数中可通过err对象捕获错误信息.
    • 如果文件不存在，自动创建一个文件。如果已存在，会覆盖原有的数据。

    writeFileSync

    fs.writeFileSync('文件地址', 数据);   //同步
    • 同步方法如果出现错误，程序会停止，后面代码不运行了。
    • 解决办法:可通过try catch结构捕获同步写法的错误

    相同点：

    如果文件名不存在，会自动新建一一个文件. 如果文件已存在，覆盖掉原本的数据。

    readFile

     fs.readFile('文件地址', function(err,data){ }) //异步
    • 错误显示在err里面
    • 获取的数据存在data里，需要使用.toString()把缓冲区的数据显示出来。

    readFileSync

    const data = fs.readFileSync('文件地址');   //同步
    • 以返回值的方式获取到数据

    path模块（处理路径）

    const path = require('path');

    join()

    • 处理路径拼接的情况更加稳妥，哪怕路径的书写习惯不一样，也能正确拼接

    basename()

    • 获取文件全名

    extname()

    • 获取文件的后缀名
    • 如果路径不是包含文件名，后缀名就为空

    http模块（创建服务器）

    const http = require('http');

    1. 导入http模块

    2. 创建一个web服务

    js const app = http.createServer();

    1. 注册一个request 事件

    js app.on('request', function(request , response){ })

    • 主要用于监听浏览器(客户端)的请求

    • 由浏览器访问的时候触发该事件

    • 监听端口

    js app.listen('端口号', function(){ })

    express

    中间件

    1. 下包

    2. 按需导入

    3. 启动中间件

    js app.use(......)

    概念

    就是请求和响应的中间做一些处理。

    app.get 和 app.post 都是基于 app.use() 封装的。

    body-parser

    • 用于处理application/x-www-form-urlencoded格式参数的post请求。

    multer

    • 用于处理form-data格式的post请求

    自己写的中间件

    app.use((request, response, next)=>{      });

    注意点

    • 在app.use() 中不要写 response.send(); ，否则其他接口 response.send() 就凉凉了。
    • 书写 app.use() 的时候，回调函数中都传入三个参数( request, response, next)，并调用 next() 函数。

    托管静态资源

    app.use('/', express.static(path.join(__dirname, 'www')))

    注意事项：无法访问 www 之外的文件，安全性问题。

    跨源资源共享 CORS

    也是后端的工作。

    解决方案：

    在后端的响应报文中，添加 Access-Control-Allow-Origin : * 。

    Access 访问，Control - 控制，Allow - 允许，Origin - 源。

    * 代表任意源否允许访问，协议，域 和 端口都不做限制。

    自己写中间件实现跨域

    统一在所有的响应报文中添加 Access-Control-Allow-Origin : *

    response.setHeader('Access-Control-Allow-Origin', '*');

    封装(了解)

    • 可以把路由的所有回调函数用一个对象进行管理。

    • 这个对象还可以独立出一个js文件。

    每个文件都是独立作用域

    暴露对象:

    
module.exports = 对象;    //会被覆盖  //就像window.属性 = 内部对象
//或者
exports.xxx = 对象;
    • 用require()语法导入这个js文件。

    另外的文件使用要导入

    const 接收的对象 = require('文件的地址');
    • express常用的方法支持链式编程。可以将 get() 和 post() 方法用链式编程连接起来

    管理路由

    const express = require('express');
const hero = express.Router();
//app.use() 启用 hero 路由
app.use('/hero/v1',hero);
//改变了接口地址
hero.get('/list',function(){ });    //地址为: 127.0.0.1:端口/hero/v1/list

    关于异步

    promise

    promise是一个对象，对象和函数的区别就是对象可以保存状态

    let p = new Promise((resolve,reject) => {
    //...
    resolve('success')
});

p.then(result => {
    console.log(result);//success
});

    async,await

    1. async表示这是一个async函数， await只能用在这个函数里面。

    2. await表示在这里等待promise返回结果后，再继续执行。

    3. await后面一般是接上promise对象

    原来的写法

    function sleep(wait) {
    return new Promise((res,rej) => {
        setTimeout(() => {
            res(wait);
        },wait);
    });
}

/*
let p1 = sleep(100);
let p2 = sleep(200);
let p =*/

sleep(100).then(result => {
    return sleep(result + 100);
}).then(result02 => {
    return sleep(result02 + 100);
}).then(result03 => {
    console.log(result03);
})

    改成async/await写法就是

    async function demo() {
    let result01 = await sleep(100);
    //上一个await执行之后才会执行下一句
    let result02 = await sleep(result01 + 100);
    let result03 = await sleep(result02 + 100);
    // console.log(result03);
    return result03;
}

demo().then(result => {
    console.log(result);
});

    如果是reject状态，可以用try-catch捕捉

    let p = new Promise((resolve,reject) => {
    setTimeout(() => {
        reject('error');
    },1000);
});

async function demo(params) {
    try {
        let result = await p;
    }catch(e) {
        console.log(e);
    }
}
