    git

    就是一个软件，该软件的功能就是帮助我们进行代码的版本管理(代码的备份)

    使用步骤

    1.下载和安装 下载地址

    安装就非常的简单，推荐安装在c盘，如果不是c盘，可能会出现一些问题。

    安装不需要思考，只需要下一步就行。

    2.安装完成之后需要先配置你的用户信息。

    右键之后，点击 git bush here,会出现一个黑窗口，在该窗口里面通过代码来配置你的用户信息

    需要配置两个： 你的名字和你的邮箱

    git config --global user.name '你的名字'

    git config --global user.email '你的邮箱'
    • 修改的话，可以再次输入以上代码，可以实现覆盖。

    查看配置是否成功

    git config --list

    如果有配置的信息，就是成功了

    文件的状态查看

    git status

    如果出现红色，说明现在的状态有问题，状态在工作区，一般底部会有提示

    如果出现绿色，文件在暂存区

    默认的颜色，所有文件都已经添加到仓库区

    查看提交日志

    git log     //显示git仓库区的提交记录

    代码回退（了解即可）

    git reset --hard 版本号    //版本号取前七、八位即可
git reset --hard head~1     //回到上一个版本

    使用git实现版本控制

    1.在你的项目的目录下面，右键选中 git bash here , 然后输入代码

    项目初始化

    git init

    会在该目录下面生成一个隐藏的文件夹 .git ， git这个软件就是依赖于这个 .git 文件夹进行管理的，如果这个文件夹丢失了，以前保存的版本的信息就丢失了

    每当一个功能完成或者是当前的工作完成，我们就会做一次版本保存。

    仍然在当前的目录下打开 git bash here , 输入代码 

    git add .

    把所有的更改都暂存起来

    然后再次输入 

    git commit -m '当前版本的备注信息'

    此时才是真正的把所有的修改都保存起来了，才能进行代码回滚

    之后，每次有了修改想要保存，就把 git add . 和 git commit -m '备注' 重新再写一次

    这样做的目的就是对操作的历史进行保存。

    git分支

    创建分支

    git branch 分支名称

git branch dev      //创建了一个叫做dev的分支

    查看分支

    git branch      //可以查看所有的分支
    • 在当前分支的前面会有一个*符号
    • 在git中，有一个特殊指针HEAD,永远会指向当前分支

    切换分支

    git checkout 分支名称

    创建并切换分支

    git checkout -b 分支名称

    删除分支

    git branch -d 分支名称
    • 注意：不能在当前分支删除当前分支，需要切换到其他分支才能删除。
    • 注意：master分支是可以删除的，但是不推荐那么做。

    合并分支

    git merge 分支名称      //将其他分支的内容合并到当前分支

    合并的分支冲突

    不同分支操作了一个文件

    解决方法：

    根据冲突标识符，删除掉不需要的代码。再继续上传到 仓库

    远程仓库和git

    git clone

    作用：克隆远程仓库的代码到本地

    git clone 远程仓库地址

    如果有仓库的git文件可以不需要这一步

    上传远程仓库的步骤

    1. git add .
    2. git commit - m '备注'
    3. git pull
    4. git push

    上传到远程指定分支

    git push origin 分支名

    首次提交可能需要

    git push -u origin 分支名

    提交时报错

    1.使用强制push的方法：

    git push -u origin master -f

    这样会使远程修改丢失，一般是不可取的，尤其是多人协作开发的时候。

    2.push前先将远程repository修改pull下来

    git pull origin master
git push -u origin master

    3.若不想merge远程和本地修改，可以先创建新的分支：

    git branch 分支名

    然后push

    git push -u origin 分支名

    撤销commit

    HEAD^表示上一个版本号

    不删除工作区更改的代码，撤销commit，不撤销add

    git reset --soft 版本号

    不删除工作区更改的代码，撤销commit，撤销add

    git reset --mixed 版本号

    删除工作区的代码，撤销commit，撤销add()

    git reset --hard 版本号

    合并commit

    git commit --amend

    把新的内容添加到之前的commit里面

    查看操作历史

    第一个方式

    可以在vscode里面安装一个插件，该插件可以看到你做的修改

    该插件安装成功之后，会在vscode的左边多一个图标

    点击该图标，从左边可以查看操作历史

    第二个方式

    安装一个软件，这个软件是github官网提供的 —— github desktop

    操作起来会更加方便

    今天只要求把这个软件的操作学会，后期我们再把代码操作写熟练

    也是直接安装，下一步就行

    1. 当我们第一个打开，会让我们登录，登录所需的账号和密码，在前天发给大家的视频里面有注册流程。自己注册

    1-1. 如果没有注册，点击 skip ,就可以直接使用了

    1. 直接把要保管的目录拖进来。会让你选择创建 仓库

    2. 然后点击 create a repository 会创建一个新的仓库 ， 会帮你创建一个.git 文件夹，并且自动保存一次

    然后在代码里面书写功能，完成一个功能之后，可以直接在软件里面输入备注，然后提交，比自己写代码稍微容易一些

    可以从history那里查看我们实现功能所写的代码的不同
