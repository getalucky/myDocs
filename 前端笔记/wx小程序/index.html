    微信小程序

    新建项目三个注意事项

    1. 目录需要选择一个空文件夹，否则可能会影响到目录不能自动生成。
    2. APPID 是小程序的唯一标识，需要注册开发者账号，并在开发-开发设置中复制过来。
    3. 后端服务，因为不需要自己写后端，所以选择不使用云服务。

    小程序的目录结构

    传统的 web 三大标准(文件)。

    1. 结构 html
    2. 样式 css
    3. 行为 js

    微信小程序的四大标准(文件)。

    1. 结构 wxml
    2. 样式 wxss
    3. 行为 js -> 注意这里语法遵循 ECMAScript 语法，但是有些对象的 API 有差异。
    4. 配置 json -> 小程序的配置项。

    配置项

    App.json 全局配置项

    存在于项目的根目录。

    官方文档：https://developers.weixin.qq.com/miniprogram/dev/reference/configuration/app.html

    小程序根目录下的 app.json 文件用来对微信小程序进行全局配置，决定页面文件的路径（pages）、窗口表现（window）、设置网络超时时间、设置多 tab 等。

    1. json 是严格的 json 格式，注意书写规范，严格 json 不能写注释。
    2. pages 是页面路径列表，数组的第一项为默认的启动页面。
    3. window 是全局的默认窗口表现，用于设置小程序的状态栏、导航条、标题、窗口背景色。
    4. tabBar 是底部 tab 栏的表现。
    5. networkTimeout 网络超时时间。

    页面配置项

    每一个小程序页面也可以使用 .json 文件来对本页面的窗口表现进行配置。

    页面中配置项在当前页面会覆盖 app.json 的 window 中相同的配置项。

    注意事项：只有 disableScroll 是页面配置项才有，其他的配置项在全局的配置项中都能看到。

    小程序界面组成

    小程序的页面分为四部分组成：

    1. 导航栏（原生的），不同的手机，导航栏的表现会有一些平台差异。
    2. tabBar（原生的）
    3. 背景（原生的）
    4. 页面 （类比之前的浏览器页面）

    页面

    block 标签

    用于编组的空标签，block 标签上可以写 wx:for wx:if 等指令。

    类似 Vue 中的 template 标签。

    应用场景：渲染一个包含多节点的结构块 block最终不会变成真正的dom元素。

    条件渲染

    频繁需要显示隐藏使用hidden，如：点击频繁显示隐藏下拉菜单

    不需要频繁的显示隐藏使用wx:if，如：权限管理，没有权限的模块直接不渲染。

    小程序 hidden 工作原理。

    小程序添加了 hidden 的标签，底层是通过选择器的方式实现隐藏。

    底层通过 view[hidden]{ display: none; } 属性选择器，实现的 div 隐藏。

    既然涉及到了选择器，就会受到权重的影响，行内样式权重过高，会导致 hidden 隐藏无效。

    1. 添加了 hidden 的盒子，匹配了内置的选择器，实现隐藏。
    2. 如果 hidden 条件为 true，相当于 display: none 生效。
    3. 如果 hidden 条件为 false，属性没有了，所以 display 用回原本的值。

    v-show 原理。

    v-show 是通过行内样式修改了盒子的显示与隐藏，行内样式权限高，问题较少。

    1. 添加了 v-show 的盒子，先检查盒子本身的 display 属性的值，存起来。
    2. 如果条件为 true，就把属性值设置了 display 之前存起来的值。
    3. 如果条件为 false，就把属性值设置了 display: none;

    小程序事件

    input 组件

    <input value="666"></input>

    input 组件常用四个事件

    bindinput 键盘输入时

    bindfocus 获取焦点

    bindblur 失去焦点

    bindconfirm 注意：手机中是点击键盘右下角的完成按钮触发事件，模拟器中是按回车键。

    获取表单值

    小程序中需要通过事件对象获取

    小程序的事件处理函数也有事件对象，但是和 web 标准的事件对象很不一样。

    获取表单值需要通过 e.detail.value 获取表单值。

    <input value="666" bindconfirm="myConfirmHandle2"></input>

Page({
  myConfirmHandle2(e) {
    console.log(e);
    console.log( e.detail.value );
  }
})

    小程序事件传参

    小程序不能的事件绑定的时候，绑定事件的属性只能写 函数名，不能直接传参。

    <button data-index="{{ index }}" bindtap="函数名">列表 {{index}} 的删除按钮</button>

    如何传递参数到事件内部？

    1. 需要在元素上添加自定义属性。
    2. 通过 事件对象的 e.currentTarget.dataset.自定义属性名 获取。
    3. 注意参数的数据类型，如果没有写花括号的自定义属性，默认数据类型是字符串。
      deleteItem(e) {
    // e.target 代表当前点击的元素
    console.log(e.currentTarget.dataset.index);
  }

    target 和 currentTarget 如何选择

    当涉及到结构嵌套时，两种情况：

    点父，e.target 和 e.currentTarget 两个没区别； 点子，e.target 是子，而e.currentTarget 是绑定事件的元素。

    建议

    尽量使用 e.currentTarget 获取事件参数。

    获取页面栈

    getCurrentPages() 是一个全局方法，可以获取小程序页面栈情况。

    getCurrentPages()

    什么时候才会有返回箭头，页面栈中的页面两个以上的时候才会有返回箭头。

    页面栈中最多保存 10 个普通页，到达了 10 个页面后，就无法再跳转普通页了，跳转 tabBar 页不受影响。

    如何实现 tabBar 页的导航也有返回箭头。自己封装一个 tabBar。

    获取用户信息

    第一次用户点击的时候需要允许授权才能获取。

    如果用户点击了拒绝，下次再点击按钮还是会弹出授权窗口。

    授权过一次后，就不再弹出了。

    // wxml
<button open-type="getUserInfo" bindgetuserinfo="getMyInfo">获取用户信息</button>

// js
Page({
    getMyInfo(e){
        const { userInfo } = e.detail;
    }
})

    打开页面发生了什么事

    第一次打开页面会自动执行以下生命周期函数。

    1. 监听页面加载 onLoad

    2. 页面显示 onShow

    3. 页面初次渲染完成 onReady

    页面隐藏 onHide

    • 右上角最小化
    • 跳转到普通页

    注意点：页面隐藏和页面显示这两个生命周期函数在一个页面中可触发多次。

    页面卸载 onUnload

    • 普通页 出 页面栈 的时候就是卸载。
    • 普通页 跳转到 tabBar 页，所有普通页都出栈。
    • 当前普通页，点击右上角返回，当前普通页也要出栈。

    注意点：

    tabBar 页一般不会被卸载的，tabBar 切换要求高性能，你看不到 tabBar 页的时候，其实都是被隐藏了。

    onUnload 注意大小写。

    属性 类型 默认值 必填 说明 最低版本
    onLaunch function 否 生命周期回调——监听小程序初始化。
    onShow function 否 生命周期回调——监听小程序启动或切前台。
    onHide function 否 生命周期回调——监听小程序切后台。

    全局的 onShow 和 onHide 主要是在小程序最小化（切后台）和再次进入（切前台）的时候触发。

    页面的的切换不会触发全局 app.js 中的 onShow 和 onHide 。

    引入组件和注册组件

    小程序中引入和注册组件两个步骤，都是在 json 配置文件中完成的。

    1. 编写子组件

    2. 在子组件的json文件中，将该文件声明为组件

    { "component": true }

    1. 在需要引入的父组件的json文件中，在usingComponents填写引入组件的组件名以及路径

    "usingComponents": { "tab-bar": "../../components/tabBar/tabBar" }

    1. 在父组件中，直接引入即可

    <tab-bar currentpage="index"></tab-bar>

    使用组件

    1. 单双标签写法都可以，但是注意标签结束。
    2. 组件名称区别大小写，注册的时候要注意。
    <!-- 单标签写法 -->
<search />

<!-- 双标签写法 -->
<search></search>

    选择器注意事项

    建议

    尽量不要直接使用 单个标签选择器，建议添加样式的时候通过类选择器。

    原因

    1. 小程序的样式自动划分作用域，但是作用域只对类名生效。
2. 标签选择器会造成样式污染，组件里面的标签选择器会影响到页面的标签选择器。

    小程序发送网络请求

    1. 小程序中没有 XHR 对象，所以不要写 new XMLHttpRequest() 。
    2. 小程序是通过 wx.request( ) API 发起 HTTPS 网络请求（不能发 http 请求）。

    小程序的 request 合法域名添加

    1. 微信开发者工具中，对不合法的域名请求会有报错提示。
    2. 添加 request 合法域名需要登录微信小程序的后台管理系统页面添加。

    合法域名要求

    1. 必须是 https 的请求，需要证书认证。
    2. 服务器域名需经过 ICP 备案，需要拍照，签保证书等流程。
    3. 不支持 IP 地址，需要买域名。

    小程序入口函数的 this 指向

    Page() 入口函数

    在 Page() 入口函数中，this 就是当前页面的实例对象。

    了解即可：getCurrentPages() 页面栈数据的最后一项其实就是当前页面的实例。

    页面数据和自定义数据

    Page({
    // 页面数据 - 和视图更新相关的数据
    data: {

    },
    // 自定义数据 - 这个数据不直接用于页面渲染，所以不写到 data 中
    aaa: 11     
})

    使用规范

    小程序中，不要把什么数据都放到 data 中，会导致页面性能降低，data 只放和页面渲染相关的数据。

    页面渲染不相干的数据，就写成自定义数据，如：分页的页码如果不需要显示到页面中，就写成自定义数据。

    小程序本地存储

    小程序的本地存储分为同步写法和异步写法两种。

    1. 同步写法和异步写法都可以实现相同功能。
    同步 异步
    运行顺序 从上往下运行，更方便阅读 异步有回调函数，什么时候结束是不知道的
    获取数据 通过返回值获取值 在回调函数中获取数据
    错误捕获 try catch 捕获错误，防止代码堵塞 在回调函数中捕获数据

    同步获取写法

    try {
  // 获取到数据
  var value = wx.getStorageSync('cate');
  // 把数据拿来使用
  if (value) {

  }
} catch (e) {
  // 如果上面代码出错了，这里捕获
}

    异步获取写法

    wx.getStorage({
    // 键名称作为参数
    key: 'cate',
    success:(res)=>{
        // 值在 res 中
    }
});

    CSS3 新增功能

    vh vw

    vh     视口高度相对单位，共 100 份，100 vh 就是占满屏幕高。
vw     视口宽度相对单位，共 100 份，100 vw 就是占满屏幕宽。

    calc()

    calc() 是 CSS3 提供的计算功能，小程序和网页中都支持，允许不同单位的运算。

    height: calc(100vh - 100rpx);

    坑点：运算符号两边至少打一个空格。

    currentColor

    颜色关键字，当前文字颜色。

    view{
    color: #0094ff;
    border: 2rpx solide currentColor;
}

    CSS3 变量

    设计稿，往往会有一定的设计规范，有些颜色是常用的，可以用 CSS 变量定义起来。

    CSS3 变量不仅可用于存放颜色，还可以存放其他 CSS 属性值。

    注意：CSS3 的变量按页面的结构分作用域。

    // 定义变量
page{
    // 颜色1
    --primaryColor: '#0094ff';
    // 颜色2
    --lightColor: '#b2b2b2';
    // 颜色3
    --lightColor: '#888';
    // 颜色4
    --semiColor: '#353535';
}


// 使用变量
view{
    color: var(--lightColor);
    background-color: var(--primaryColor);
}
